## Themes

### 1. Global shortcuts — new central registry

New module `global_shortcuts.rs` unifies registration for quick-TTS presets and soundboard slots. It detects shortcut conflicts, sets `shortcut_conflict` flags on soundboard slots, and reloads all bindings on settings changes. **Risk:** overlapping bindings between voice-profile shortcuts and soundboard defaults (`Ctrl+Shift+N`) are flagged but resolution is left to the user; worth smoke-testing after upgrade.

### 2. HTTP API surface expanded

`docs/API.md` documents 11 new plugin/soundboard routes under `/plugins/*`, including slot assign/play/stream. The local server remains `127.0.0.1:8765` with CORS `allow_origin(Any)` (unchanged pattern). **Notable:** `PUT /plugins/soundboard/slots/{index}` accepts arbitrary `file_path` values; Rust copies the file into `%APPDATA%\TTS_hub\plugins\soundboard\` — any local process that can reach the API can trigger file reads/copies.

### 3. Cursor / agent workflow

- Skill docs (`SKILL.md`, `README.md`, `CURSOR_SKILL.md`) clarify that voice comes from **Integracja Cursor** in the app when `prefer_app_config: true`.
- `speak-summary.ps1` merges provider-specific fields from `GET /cursor/config` for MiniMax, Google, and Voice Box (previously MiniMax model/voice were intentionally excluded from app override).
- Soundboard section added to the skill so agents can assign/trigger clips without UI.

### 4. Dev ergonomics

New scripts under `scripts/dev/`: `tts-hub-dev-start.bat`, `tts-hub-dev-stop.bat`, `stop-tauri-dev.ps1`, `create-shortcuts.ps1` — documented in `scripts/dev/README.md`.

## Security & reliability notes

- **No auth on localhost API** — Existing design; new soundboard endpoints inherit the same trust model (any local caller).
- **`file_path` slot assignment** — Copies user-specified paths into app storage; no path-sandbox beyond existence check at copy time.
- **Windows-centric audio path** — Output-device selection and WebView permission work are gated on Windows APIs; Linux/macOS behavior should be verified separately (`global_shortcuts` is no-op on mobile; audio output UI may degrade gracefully).
